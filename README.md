## Part 1: Imaging Technique Inspiration

### Perlin Noise Flow Field – Generative Particle Art

**Inspiration Source:** *Flow Field* generative artworks by artists such as Tyler Hobbs and Daniel Shiffman (The Coding Train).

**Description:**

The technique I find most inspiring is **Perlin Noise Flow Fields** — a generative art method where thousands of particles move along invisible vector paths shaped by smooth, organic noise. Unlike random movement, Perlin noise creates natural, flowing curves that feel almost alive. I want to incorporate this into my project because it produces visually rich, hypnotic animations from minimal input data. It aligns perfectly with the p5.js assignment requirements, as p5.js natively supports `noise()` for Perlin noise generation. The emergent complexity from simple rules makes it both technically interesting and aesthetically compelling.

## Part 2: Coding Technique Exploration

### p5.js `noise()` Function with Vector Fields

**Technique:** Using p5.js's built-in `noise(x, y)` function to generate a 2D grid of angle vectors, then moving particles along those vectors each frame to simulate flowing, organic motion.

**Description:**

The noise() function in p5.js generates smooth, naturally flowing values — unlike random(), which jumps unpredictably. By feeding each particle's position into noise(), we get a direction angle that feels organic and fluid. Particles then move along these angles, tracing beautiful curving paths across the canvas. As a small time offset is added each frame, the entire field slowly shifts and evolves, creating a living, breathing animation. This directly supports the flow field aesthetic from Part 1 — smooth, hypnotic motion emerging from a simple mathematical rule.
